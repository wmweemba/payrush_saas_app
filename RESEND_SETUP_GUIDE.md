# Resend.com Email Service Setup Guide

## üöÄ **Quick Setup Instructions**

### **Step 1: Create Resend.com Account**

1. **Go to Resend.com**: Visit [https://resend.com](https://resend.com)
2. **Sign Up**: Click "Get Started" and create account with:
   - **Email**: Your business email address
   - **Password**: Strong password
   - **Company Name**: Your business name (e.g., "PayRush")

### **Step 2: Verify Your Domain (Recommended)**

1. **Add Domain**: In Resend dashboard, go to "Domains" ‚Üí "Add Domain"
2. **Enter Your Domain**: Add your business domain (e.g., `yourdomain.com`)
3. **DNS Configuration**: Add the following DNS records to your domain:
   ```
   Type: TXT
   Name: @
   Value: [Resend will provide this value]
   
   Type: CNAME  
   Name: resend._domainkey
   Value: [Resend will provide this value]
   
   Type: MX
   Name: @
   Value: feedback-smtp.resend.com
   Priority: 10
   ```
4. **Verify Domain**: Click "Verify Domain" (may take up to 48 hours)

### **Step 3: Get API Key**

1. **Navigate to API Keys**: Go to "API Keys" section in Resend dashboard
2. **Create New API Key**: 
   - **Name**: "PayRush Production" or "PayRush Development"
   - **Permissions**: "Sending access" (default)
   - **Domain**: Select your verified domain OR use `onboard.resend.dev` for testing
3. **Copy API Key**: Copy the generated API key (starts with `re_`)

Please provide me with:

## üìß **Email Configuration Options**

### **Option A: Use Verified Domain (Recommended for Production)**
- **From Email**: `invoices@yourdomain.com` or `noreply@yourdomain.com`
- **Benefits**: Professional appearance, better deliverability, custom branding
- **Setup Time**: 24-48 hours for domain verification

### **Option B: Use Resend Test Domain (Quick Start)**
- **From Email**: `onboarding@resend.dev`
- **Benefits**: Immediate setup, no DNS configuration needed
- **Limitations**: Generic sender address, may affect deliverability
